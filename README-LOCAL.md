# 현장별 매출·손익 관리 시스템 (로컬 실행 버전)

Node.js만 설치되어 있으면 별도 의존성 설치 없이 바로 실행됩니다.

## 🚀 실행 방법

### 1. Node.js 설치 확인
```bash
node --version
```
설치되어 있지 않다면 [nodejs.org](https://nodejs.org)에서 다운로드 (v16 이상 권장).

### 2. 서버 실행
프로젝트 폴더에서 아래 명령어 중 하나 실행:

```bash
node code.gs
# 또는
node code.js
```

### 3. 브라우저로 접속
```
http://localhost:3000
```

### 4. 종료
터미널에서 `Ctrl + C`

## 📁 파일 구성

```
construction-pnl-local/
├── code.gs           # 백엔드 서버 (Node.js, code.js와 동일)
├── code.js           # code.gs와 같은 파일 (확장자만 다름)
├── index.html        # 프론트엔드 SPA
├── README.md         # 이 파일
└── data/
    └── db.json       # 자동 생성되는 데이터 파일
```

## 💾 데이터 저장

모든 데이터는 `./data/db.json` 파일에 자동 저장됩니다. 이 파일을 백업하면 데이터 백업이 됩니다.

## 🎯 주요 기능

- **대시보드**: KPI 카드, 월별 추이, 현장별 비교
- **현장 관리**: 현장 등록/수정/삭제 (CRUD)
- **월별 실적 입력**: 모든 현장의 월별 매출/원가 일괄 입력
- **현장 상세**: 12개월 그리드 편집, 준공추정 입력, 차트
- **계획 대비 분석**: 계획 vs 실적 vs 추정 비교

### 자동 계산 항목
- 매출이익 = 매출 - 매출원가
- 매출원가율 = 매출원가 / 매출 × 100
- 누계 = 1월부터 선택월까지 합계
- 잔여 = 연간계획 - 누계매출
- 달성률 = 누계매출 / 연간계획 × 100

### 입력 검증
- 매출이 0일 때 매출원가가 0보다 크면 차단
- 같은 현장에 같은 년월 데이터 중복 입력 차단

## 🔧 환경 변수

- `PORT` — 서버 포트 변경 (기본 3000)
  ```bash
  PORT=8080 node code.gs
  ```

## 📊 빠른 시작

처음 접속하면 비어있는 대시보드가 나옵니다. 좌측 하단 **"샘플 데이터 추가"** 버튼을 눌러 테스트 데이터를 넣어보세요.

## 🛠 트러블슈팅

| 증상 | 해결 |
|---|---|
| `Error: listen EADDRINUSE` | 3000 포트가 이미 사용 중. `PORT=8080 node code.gs`로 다른 포트 사용 |
| `index.html을 찾을 수 없습니다` | `code.gs`와 `index.html`이 같은 폴더에 있는지 확인 |
| 데이터 초기화하고 싶음 | `data/db.json` 파일 삭제 후 재실행 |
| 한글 깨짐 | 터미널 인코딩이 UTF-8인지 확인 |

## 📦 다른 PC로 이동

폴더 전체를 복사하면 됩니다. `data/db.json`까지 복사하면 데이터도 그대로 이동됩니다.
